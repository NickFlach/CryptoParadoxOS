1. Understanding the Data You Need
Before writing any code, decide which repository metrics will be most useful for your research or modeling. Common fields include:

owner: the GitHub username or organization name
repo: the repository name
description: a short text describing the repo’s purpose
language: the primary coding language (e.g., Python, Solidity, JavaScript)
stars: the number of GitHub stars (often a rough popularity signal)
forks: the number of forks
open_issues: the number of open issues
watchers_count: watchers on the repo (less used these days, but still trackable)
license: the license type (MIT, GPL, etc.)
last_push_date: the last push or commit date (can indicate recent activity)
created_at: date the repository was created
updated_at: date the repo was last updated
size: approximate size of the repo (in kilobytes)
You could also incorporate advanced metrics such as pull request counts, contributor stats, release tags, or commit frequency, but these require more complex queries.

2. Using Python to Gather GitHub Data
Below is a sample Python script that uses the requests library to query the GitHub REST API. Alternatively, you can use PyGithub if you prefer an object-oriented approach.

import requests
import csv
import os

# If you prefer, store your token in an environment variable:
# export GITHUB_TOKEN="your_token_here"
GITHUB_TOKEN = os.getenv("GITHUB_TOKEN", "YOUR_GITHUB_TOKEN")

# Create a reusable headers dict for all API requests
headers = {
    "Authorization": f"Bearer {GITHUB_TOKEN}",
    "Accept": "application/vnd.github.v3+json"
}

def fetch_repo_info(owner, repo):
    """
    Fetch basic information about a single GitHub repository using the REST API.
    """
    url = f"https://api.github.com/repos/{owner}/{repo}"
    
    response = requests.get(url, headers=headers)
    
    if response.status_code == 200:
        data = response.json()
        repo_info = {
            "owner": owner,
            "repo_name": repo,
            "description": data.get("description") or "",
            "language": data.get("language") or "",
            "stars": data.get("stargazers_count", 0),
            "forks": data.get("forks_count", 0),
            "open_issues": data.get("open_issues_count", 0),
            "watchers_count": data.get("watchers_count", 0),
            "license": data["license"]["spdx_id"] if data.get("license") else "None",
            "created_at": data.get("created_at"),
            "updated_at": data.get("updated_at"),
            "last_push_date": data.get("pushed_at"),
            "size": data.get("size", 0)
        }
        return repo_info
    else:
        print(f"Failed to fetch {owner}/{repo}: HTTP {response.status_code}")
        return None

def main():
    # EDIT THIS LIST: Repositories you want to query
    # Format: list of (owner, repo_name)
    repos_to_fetch = [
        ("ethereum", "go-ethereum"),
        ("ethereum", "solidity"),
        ("ethereum", "py-evm"),
        # Add more pairs relevant to your research...
    ]
    
    # Name of the CSV file you want to create
    output_csv = "github_data.csv"
    
    # Fieldnames for your CSV columns
    fieldnames = [
        "owner", "repo_name", "description", "language", "stars",
        "forks", "open_issues", "watchers_count", "license",
        "created_at", "updated_at", "last_push_date", "size"
    ]
    
    # Open the CSV file in write mode
    with open(output_csv, mode="w", newline="", encoding="utf-8") as f:
        writer = csv.DictWriter(f, fieldnames=fieldnames)
        writer.writeheader()  # Write the column headers
        
        for (owner, repo) in repos_to_fetch:
            repo_info = fetch_repo_info(owner, repo)
            if repo_info:
                writer.writerow(repo_info)
    
    print(f"Data written to {output_csv}")

if __name__ == "__main__":
    main()

5. Further Enhancements
Fetching multiple pages of repositories: If you want to gather an entire organization’s repositories automatically, you can query the GitHub API’s “List organization repositories” endpoint (e.g., GET /orgs/{org}/repos). This returns a paginated list of all repos under that org. You’d then loop over each page until you collect all repo names.
Advanced stats: For more granular data (e.g., commit counts, contributor stats, or release info), explore additional GitHub endpoints like Commits API or Pull Requests API.
Rate limiting: Authenticated requests get higher rate limits (usually 5,000 requests/hour per user/token). If you query massive sets of data, you may need to handle pagination carefully or include a small delay between requests to avoid hitting the limit.
Automation: If you want to run this data collection regularly, consider a CI/CD pipeline or cron job that updates the CSV daily or weekly and pushes changes to your repository.
Data cleansing: Once you have the CSV, do some QA checks for missing values, inconsistent data, or duplicates. This is especially important if you integrate data from multiple sources (e.g., combining GitHub data with your own custom dataset).
